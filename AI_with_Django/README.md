# House Price Predictor

A Django web application that estimates a home's price from its area, bedroom count, bathroom count, and age. The prediction is generated by a trained machine-learning model and presented through a responsive, polished web interface.

## Highlights

- Instant house-price estimates from a saved ML model.
- Django form handling with CSRF protection.
- Responsive prediction page for desktop and mobile.
- CSS-only 3D/neubrutalist visual design: solid colours, bold borders, and layered shadows—no gradients, images, or front-end libraries.
- Clear form labels, browser-side input validation, and a prominent prediction result.

## Preview of the experience

The landing page pairs a concise introduction and a small CSS-built house illustration with a property-details form. After submission, the model estimate appears in a highlighted result panel while the form remains available for another calculation.

## Requirements

- Python 3.10 or newer
- Django
- joblib
- scikit-learn (required to load the serialized model)

## Run locally

1. Create and activate a virtual environment.

   ```powershell
   python -m venv .venv
   .\.venv\Scripts\Activate.ps1
   ```

2. Install the dependencies.

   ```powershell
   pip install django joblib scikit-learn
   ```

3. Start the Django development server from the `houseapp` directory.

   ```powershell
   cd houseapp
   python manage.py runserver
   ```

4. Open `http://127.0.0.1:8000/` in your browser.

## How to use it

Fill in each property detail and select **Calculate my estimate**.

| Input | Description |
| --- | --- |
| Built-up area | Total property area in square feet. |
| Bedrooms | Number of bedrooms in the property. |
| Bathrooms | Number of bathrooms in the property. |
| Property age | Age of the home in years. |

The app sends these four values to the trained model and displays the estimated price in Indian rupees. This is a model-based indication only, not a formal property valuation.

## How it works

1. `houseprediction/views.py` loads `house_model.pkl` using `joblib`.
2. The root URL calls `predict_fun`.
3. On a POST request, Django reads `area`, `bedrooms`, `bathrooms`, and `age` from the form.
4. Those values are passed to the model in the feature order it expects.
5. The prediction is returned to `prediction.html` as `pred` and rendered in the estimate panel.

## Design notes

The interface deliberately uses a non-gradient 3D style. The depth comes from offset shadows, strong outlines, stacked card surfaces, and a CSS-only house illustration. It is fully self-contained in `houseapp/houseprediction/templates/prediction.html`, making it straightforward to customize without adding front-end build tools.

For a reusable instruction to apply this design approach to another Django prediction project, see [prediction_page_design_prompt.txt](prediction_page_design_prompt.txt).

## Important note

The app currently loads the trained model from an absolute path in `houseprediction/views.py`. If you move or clone the project to a different location, update that path or refactor it to use a project-relative path before running the application.
